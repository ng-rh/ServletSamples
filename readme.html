Servlet 3.0 Samples

Servlet 3.0 Samples demonstrates some new Servlet 3.0 functionality. See how to programmatically add servlets, configure web-fragments, use an asynchronous servlet, or use a ServletContainerInitializer.

Here is what you should know about this sample before proceeding.

Time required to set up and configure this sample

5 minutes

Prerequisites for use

  1. WebSphere Application Server. This sample was tested most recently with Version 8.0 of the product.

About this task

The following key phrases are associated with this sample: Servlet 3.0, web-fragment, asynchronous servlet, ServletContainerInitializer

See the following more detailed description of the sample:
SampleProgrammatic.war

This module shows how to programmaticaly add servlets, filters, and listeners, or all three objects.

For more information, see Servlet 3.0 programmatic configuration.

SampleAsyncContext.war

This module shows how to use an asynchronous servlet to process a long running request and then dispatch back to write out the response.

For more information, see Java Servlet 3.0 considerations.

SampleWebFragment.war

This module shows how to include a JAR file that contains a web-fragment.xml file.

For more information, see Web fragments.

SampleServletContainerInitializer.war

This module shows how to use a ServletContainerInitializer to help initialize the application.

For more information, see ServletContainerInitializer.

Procedure

    Re-build the sample (optional)

    1. Go to the directory where you extracted the sample.
    2. Then, run [PathOfWasInstall]\bin\ws_ant.[bat|sh] -f build.xml build

    Install the sample

    1. Go to the directory where you extracted the sample.
    2. Then, run [PathOfWasInstall]\bin\ws_ant.[bat|sh] -f build.xml installAndStart -Duser=username -Dpassword=pass -DcellName=cell -DnodeName=node -DserverName=server
    3. If the above command fails, you should make sure that the server is up and running. If it is and you have more than one profile installed, you may also want to verify that the command is using the profile that you expect it to use. Either modify the default profile or use the profile's bin directory to run the ws_ant command.

    Run the sample

    1. Open the following URLs in a browser:
      • Hit http://server:port/SampleProgrammatic/Servlet1
      • Hit http://server:port/SampleProgrammatic/Servlet2
      • Hit http://server:port/SampleProgrammatic/Servlet3
      • Hit http://server:port/SampleAsyncContext/MainServlet
      • Hit http://server:port/SampleWebFragment/AnnotatedServletInWebFragment
      • Hit http://server:port/SampleWebFragment/DeclaredServletInWebFragment
      • Hit http://server:port/SampleWebFragment/DeclaredServletInWebXML
      • Hit http://server:port/SampleServletContainerInitializer/AnnotatedServlet

    Uninstall the sample

    1. Go to the directory where you extracted the sample.
    2. Then, run [PathOfWasInstall]\bin\ws_ant.[bat|sh] -f build.xml stopAndUninstall -Duser=username -Dpassword=pass -DcellName=cell -DnodeName=node -DserverName=server
    3. If the above command fails, you should make sure that the server is up and running. If it is and you have more than one profile installed, you may also want to verify that the command is using the profile that you expect it to use. Either modify the default profile or use the profile's bin directory to run the ws_ant command.
